
# AI-Assisted Security Reporting

## Overview
This project demonstrates how AI can support security operations by helping analyse Nmap and Nessus findings and convert them into structured incident reports.

## Purpose
In security operations, reviewing scan outputs and writing weekly reports can be time-consuming. This project shows a simple workflow where raw findings are reviewed, prioritised, and converted into clear reports with risk levels and remediation guidance.

## Tools Used
- Nmap
- Nessus
- ChatGPT
- Microsoft Sentinel
- Splunk
- IBM QRadar

## Workflow
1. Collect scan results from Nmap or Nessus.
2. Review open ports, exposed services, vulnerabilities, and misconfigurations.
3. Use AI assistance to structure findings into a clear report.
4. Validate the output manually.
5. Prepare final incident or vulnerability report.

## Example Report Format
- Summary
- Key Findings
- Risk Level
- Evidence
- Recommended Remediation
- Status / Follow-up
